# Rúbrica de Evaluación
## Introducción a la Programación

Este documento define la **rúbrica oficial de evaluación** del Cuatrimestre del curso de Programación en C++ orientado a videojuegos.

El objetivo de esta rúbrica es evaluar **pensamiento computacional, claridad, estructura y aplicación de buenas prácticas**, no únicamente si el programa ejecuta correctamente.

Esta rúbrica se aplica a:
- Prácticas
- Proyectos
- Ejercicios evaluables
- Super Challenge

---

## CRITERIOS DE EVALUACIÓN

---

### 1. Lógica y funcionamiento (30%)
- El programa cumple con los requisitos solicitados
- El flujo lógico es correcto
- No presenta errores críticos en ejecución

---

### 2. Estructura del código (20%)
- Uso adecuado de estructuras de control
- Uso correcto de funciones cuando el ejercicio lo requiere
- Separación clara de responsabilidades

---

### 3. Buenas prácticas de programación (20%)
- Código escrito completamente en inglés
- Uso de nombres descriptivos
- Aplicación correcta de camelCase, PascalCase y SNAKE_CASE
- Comentarios claros y no redundantes

---

### 4. Claridad y legibilidad (15%)
- Código fácil de leer
- Indentación consistente
- Organización lógica del archivo

---

### 5. Documentación (10%)
- Uso correcto de archivos Markdown cuando aplique
- README claro y comprensible
- Explicación adecuada del funcionamiento del programa

---

### 6. Uso de GitHub (5%)
- Repositorio correctamente organizado
- Commits claros y coherentes

---

## ESCALA DE DESEMPEÑO

- **Excelente (90–100):** Código claro, bien estructurado, cumple todos los criterios y aplica consistentemente las buenas prácticas
- **Bueno (70–89):** Cumple los requisitos funcionales y la mayoría de los criterios, con áreas claras de mejora
- **Suficiente (60–79):** El programa funciona, pero presenta deficiencias importantes en estructura, claridad o buenas prácticas
- **Insuficiente (<60):** No cumple los requisitos mínimos o el código es ilegible

---

## NOTAS IMPORTANTES

- Un programa que funciona pero no sigue buenas prácticas **no puede obtener calificación máxima**.
- El plagio o la entrega de código sin comprensión demostrable implica calificación no aprobatoria.
- La evaluación considera tanto el **resultado** como el **proceso de desarrollo**.
- Cuando existan dudas razonables sobre la autoría o el dominio del trabajo, el estudiante deberá **explicar, defender y realizar ajustes al código** durante una revisión guiada para demostrar comprensión.

Este documento es oficial para el Cuatrimestre: *Introducción a la Programación* y sirve como referencia durante todo el curso.
